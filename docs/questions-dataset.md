# 🍪 두바이 쫀득 쿠키 테스트: 80개 랜덤 문항 데이터셋

## 📌 데이터셋 개요

- **총 문항 수**: 80개
- **구조**: 20개 단계 × 4개 변주 = 80개
- **랜덤 전략**: 사용자가 테스트를 반복할 때마다 각 단계에서 4개 변주 중 1개를 랜덤 선택
- **목적**: 재방문율 향상 및 AdSense 노출 극대화

## 💡 구현 전략

### 랜덤 선택 로직

```typescript
/**
 * 각 단계에서 4개 변주 중 1개를 랜덤으로 선택
 */
function getRandomQuestion(step: number): Question {
  const variants = ['A', 'B', 'C', 'D'];
  const randomIndex = Math.floor(Math.random() * 4);
  const selectedVariant = variants[randomIndex];

  return questionPool[step][selectedVariant];
}
```

### 재방문 최적화

- **세션별 문항 조합**: SessionStorage에 선택된 변주 저장
- **중복 방지**: 이전 테스트 이력과 비교하여 최대한 다른 변주 선택
- **A/B 테스팅**: 특정 변주의 참여율/완주율 추적

### AdSense 노출 전략

- 각 단계마다 페이지 전환 (`/test/[step]`)
- 5단계마다 중간 광고 삽입 (단계 5, 10, 15 이후)
- 로딩 페이지(결과 계산)에서 전면 광고 노출

## 📊 데이터 구조 예시

```typescript
interface QuestionVariant {
  id: string; // 예: '1-A', '1-B'
  step: number; // 1~20
  variant: 'A' | 'B' | 'C' | 'D';
  dimension: Dimension; // 'EI', 'SN', 'TF', 'JP'
  title: string; // 질문 제목 (단계명)
  option1: {
    label: string; // 옵션1 텍스트
    value: number; // EI/SN/TF/JP 중 첫 번째 알파벳에 +1
  };
  option2: {
    label: string; // 옵션2 텍스트
    value: number; // EI/SN/TF/JP 중 두 번째 알파벳에 +1
  };
}
```

### 실제 데이터 예시

```typescript
const question_1A: QuestionVariant = {

  },
```
